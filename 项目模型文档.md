# 项目模型文档

更新时间：2026-04-07

## 当前默认主链路

- 音乐生成：`ACE-Step`
- 人声分离：`Demucs`
- 音色转换：`RVC`

当前代码默认配置：

- `MUSIC_GEN_MODEL = "ace_step"`
- `VOCAL_SEPARATOR_MODEL = "demucs"`
- `VOICE_CLONE_MODE = "rvc"`
- `SVC_MODEL = "rvc"`

## 音乐生成模型

模型目录：

- `models/Ace-Step1.5`

当前使用到的模型名字：

- `Ace-Step1.5`
- `acestep-v15-turbo`
- `acestep-5Hz-lm-1.7B`
- `Qwen3-Embedding-0.6B`
- `vae`
- `acestep`

说明：

- `acestep-v15-turbo` 是主生成模型；`acestep-5Hz-lm-1.7B` 是歌词/语言模型；`Qwen3-Embedding-0.6B` 是文本嵌入模型；`vae` 是解码组件；`acestep` 是官方推理包，安装后会优先走官方推理链路

## RVC 音色转换模型

模型目录：

- `models/RVC1006Nvidia`
- `models/rvc_pretrained`
- `models/user_voices`

RVC 运行时仓库名：

- `RVC1006Nvidia`

当前检测到的官方示例音色名：

- `guanguanV1`
- `keruanV1`
- `kikiV1`
- `youzhanv2-xi`

当前默认示例音色：

- `kikiV1`

已存在的示例音色索引文件：

- `guanguanV1.index`
- `keruanV1.index`
- `kikiV1.index`

RVC 预训练依赖文件名：

- `D48k.pth`
- `G48k.pth`
- `hubert_base.pt`
- `rmvpe.pt`

说明：

- `D48k.pth` 是判别器权重；`G48k.pth` 是生成器权重；`hubert_base.pt` 是 HuBERT 内容编码器；`rmvpe.pt` 是 RMVPE 音高提取器

用户自定义音色模型放置位置：

- `models/user_voices/*.pth`
- `models/user_voices/*.index`

## 当前不是默认主链路的可选项

- `musicgen`
- `fish_speech`
- `diffsinger`
- `vits-svs`

说明：

- 这些名字在配置或脚本里有出现
- 但当前默认实际使用的主链路仍然是 `ACE-Step + Demucs + RVC`

## 仓库说明

- 模型文件默认放在根目录下的 `models/`
- 仓库已经忽略 `/models/`，所以模型权重不会被上传到 GitHub
- GitHub 仓库中保留的是代码和文档，模型需要在本地单独准备
